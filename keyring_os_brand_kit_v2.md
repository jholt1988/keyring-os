# Keyring OS — Brand Kit v2

## 1. Brand Positioning

Keyring OS is a **decision-driven operating system for property management**.

It is not:
- a dashboard
- a reporting tool
- a traditional SaaS platform

It is:
> A system that surfaces what matters, recommends actions, and enables execution.

---

## 2. Brand Attributes

- Intelligent
- Operational
- Precise
- Minimal
- Confident
- Calm under pressure

---

## 3. Voice & Tone

### Voice
- Direct
- Action-oriented
- Clear
- Non-decorative

### Tone
- Confident, not arrogant
- Helpful, not verbose
- Focused, not explanatory unless needed

### Examples

❌ “You might want to consider reviewing this payment”  
✅ “Payment overdue — send notice”

---

## 4. Visual Identity

### Core Feel
- Dark, high-contrast interface
- Subtle glow and depth
- Minimal clutter
- Spatial, not grid-heavy

### Design Principles
- Reduce visual noise
- Highlight action
- Use color for meaning, not decoration

---

## 5. Color System

### Base
- Background: Deep black / navy
- Surface: Glass / translucent dark panels

### Signals
- Cyan → system active / neutral
- Amber → attention required
- Red → critical / urgent
- Green → resolved / stable

---

## 6. Typography

- Clean sans-serif
- Tight tracking for headers
- High contrast hierarchy

### Roles
- Title → decision / object
- Label → context
- Body → minimal explanation

---

## 7. Motion Philosophy

- Fast (150–250ms)
- Intentional
- No decorative animation
- Movement should communicate:
  - focus
  - hierarchy
  - causality

---

## 8. Interaction Model

Replace:
- navigation
- dashboards

With:
- briefing
- command
- execution

---

## 9. Core UI Concepts

- Command Node (Orb)
- Radial Menu
- Decision Cards
- Signal Layers
- Context Panels

---

## 10. Brand Rule

> If it does not help the user decide or act, it does not belong.
